Angular

Introduction

Angular is a platform and framework for building client-side, single-page applications (SPAs: a web application that loads a single HTML page and dynamically updates the content of that page as the user interacts with it ) using HTML and TypeScript. Developed and maintained by Google, it provides a structured approach to building complex, dynamic web applications with a focus on modularity( breaking down a large program into smaller), reusability, and maintainability (process of modifying, updating, or adapting a software)

Difference between Angular and React

Angular is a complete framework with a structured, opinionated approach to web development. React is a JavaScript library focused on building user interfaces.

Angular is like a blueprint for building a house; it gives you a clear plan and all the tools. React is like a toolkit for building a house; it gives you great tools for specific tasks, but you have to come up with the plan yourself.

Why Angular?

Structure: It provides a consistent, built-in structure for large-scale applications, making them easier to manage and scale. TypeScript: It uses TypeScript by default, which adds type safety and helps catch errors early. "Batteries Included": Angular includes everything you need out of the box—routing, state management, and an efficient command-line interface (CLI).

Prerequisites needed for angular

Node.js & npm: Angular runs on Node.js and uses its package manager, npm, to install all the necessary dependencies. You'll need to have a recent, stable version of Node.js installed on your system. npm -> " It's the standard tool for managing JavaScript packages and is the default package manager for Node.js. "

IDE(Integrated Development Environment)(Code-Editor): You'll need a good code editor with solid support for Angular, TypeScript, and debugging. Visual Studio Code is the most popular choice as it has excellent extensions specifically for Angular.

AngularCLI :"The main work of the Angular CLI is to automate and streamline the entire development workflow". The CLI automates the creation of a new Angular project with a single command (ng new my-app). This sets up all the necessary files, configurations, and dependencies, saving you hours of manual work. It can quickly generate core building blocks like components, services, and modules using simple commands (ng generate component my-component). This ensures a consistent project structure and saves you from manually creating and linking files.The CLI is integrated with testing frameworks, allowing you to run unit and end-to-end tests with simple commands

To install AngularCLI the command is "npm install -g angular/cliversion_num" (here '-g' indicated it is installed globally )

NPM :Node Package Manager, is the world's largest software registry and a command-line tool for managing JavaScript packages.It allows developers to easily share, discover, and use pre-written code packages to speed up their development process.

Components

A Component is the basic building block of an Angular application that controls a section of the user interface.It has three parts: a class (for logic), a template (for HTML view), and styles (for design). The Component decorator tells Angular how to use it. Every app has one main component called AppComponent, and more components can be added to make the app organized and reusable.

It contains mainly 4 main parts

  • Html Component
  • css Component
  • spects.ts component
  • ts component

App Component

It's the main component in a angular file.The AppComponent is the root component of every Angular application. It is the first component that loads when the app starts and acts as the main container for the whole application. All other components are either directly or indirectly children of the AppComponent. It is defined in the app-component-ts file, and it uses an HTML template (app-component-html) and styles (app-component-css). The selector for AppComponent is usually "app-root", which is placed inside the index.html file to display the Angular app in the browser. In short, AppComponent is the entry point and backbone of an Angular application.

Type Script Based

Angular is built on TypeScript because it provides features that plain JavaScript lacks, such as type safety, classes, interfaces, and better tooling support. These features help developers catch errors early, write cleaner code, and build large, complex applications more easily and reliably.Also Helps in error detection at compile time and makes code more scalable. TypeScript also compiles down to JavaScript, so Angular apps can run in any browser.

TypeScript → JavaScript → Browser’s JS Engine → Machine Code (Binary) → CPU Execution

Angular Project Structure

main-ts → entry point that bootstraps the app

index-html → root HTML file where app-root is placed

angular-json → config file for build & project settings

package-json → dependencies and npm scripts

Structure of an Angular Project

When we create a new Angular project using the Angular CLI command ng new project-name, it automatically generates a standard folder structure. This structure helps in organizing files and makes the development process easier and more scalable.

Main Folders and Files

  • e2e/ → Contains end-to-end test files to test the application flow as a whole.
  • node_modules/ → Contains all project dependencies and packages installed through npm.
  • src/ → The main source folder where the actual application code resides.
    • app/ → Contains the core of the application such as modules, components, and services.
      • app.component.ts → Logic of the root component.
      • app.component.html → Template (UI) of the root component.
      • app.component.css → Styling of the root component.
      • app.module.ts → Root module that bootstraps the application.
    • assets/ → Stores static files like images, icons, and JSON data.
    • environments/ → Contains environment-specific configuration files (development, production).
    • main.ts → The entry point of the Angular app. It bootstraps the root module (AppModule).
    • index.html → The main HTML file loaded in the browser. The Angular app gets displayed inside the <app-root> selector here.
    • styles.css → Global stylesheet for the entire project.
    • polyfills.ts → Provides compatibility for older browsers.

Configuration Files

  • angular.json → Defines project configuration (build settings, styles, scripts, assets).
  • package.json → Lists project dependencies, scripts, and npm commands.
  • tsconfig.json → TypeScript configuration file.
  • karma.conf.js → Configuration file for running unit tests.
  • tslint.json (or eslint.json in newer versions) → Defines linting rules for code quality.

This structure ensures that Angular applications are modular, maintainable, and scalable. Developers can easily add new components, services, and modules without breaking the organization of the project.
