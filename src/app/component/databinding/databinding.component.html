Data Binding

Definition of Data Binding

Data binding is a technique that connects the data in the TypeScript (component) with the HTML (view), so that any change in one is immediately reflected in the other.It automates the UI update process, making your code simpler, more predictable, and easier to maintain. You don't have to manually write code to push and pull data between your HTML and your logic.

Types of Data Binding

There are mainly two types of data bindings

  • One way Data Binding
  • Two way Data Binding

One Way Data Binding

One way data binding consists of 3 types

  • Interpolation
  • Property Binding
  • Event Binding

Interpolation

Interpolation in Angular is a type of one-way data binding that allows you to display dynamic data from the component (TypeScript code) into the view (HTML template).

It uses the (double curly braces) syntax. Whatever value is written inside the curly braces is evaluated by Angular and the result is displayed as text in the HTML.It can display variables, expressions, method return values, and even perform simple calculations. It cannot change values in the component, it only shows them in the template.

The data flows in only one direction: from the component → to the view.

Interpolation = one-way data binding (component → view) using double curly braces.

Property Binding

Property Binding is a feature in Angular that lets you bind component data to HTML element properties. It updates the element’s property value whenever the component’s data changes, ensuring that the UI always shows the latest data.

Example

  • <img [src]="imageUrl" >

In this example, the src property of the image will always display the value of imageUrl from the component.

ImgUrl that we are using should be declared in TS file with type string. ImgUrl stores the image address, while we are doing bind in output we are not getting the img link instead we are getting the direct image i.e, here applies the property binding.

Property Binding → one-way (component → element property)

Property Binding = one-way data binding (component → view) by binding data to element properties using square brackets [ ]

Event Binding

A technique in Angular that allows the view (HTML) to listen to events (like clicks, input, or mouse movements) and call a method in the component when the event occurs. It’s a one-way binding from view → component.

In this a 'event' is first declared in HTML and then it is called in TS component and the required logic will be written there, when the event happens the changes will appear in UI

It connects a user action in the template to a function in the component.

Event Binding = one-way data binding (HTML → TS component) using parentheses to listen to events, e.g., (click)="methodName()".

Two Way Data Binding

Two-way Data Binding is a mechanism that allows synchronization between the component and the view. When the data in the component changes, the view updates automatically, and when the user changes the data in the view (e.g., typing in an input), the component’s property is updated as well. It combines property binding and event binding into a single step using [( )] syntax called banana-in-a-box.

This can be achived bu using "ngModel" key word in input tag placing in [( )] and assining the value which is declared in TS that have to be modified while input tag is getting modified

It uses the [( )] syntax, called banana-in-a-box.

Data Binding is a mechanism to coordinate data between the component and the view in Angular. It can be one-way (component → view or view → component) or two-way (both ways), allowing automatic updates when data changes.
