Directives

Defintion of Directives

Directives in Angular are special instructions that add custom behavior to elements in the DOM. They are used to change the structure, appearance, or behavior of HTML elements. Simply, A directive is a class that adds new behavior to elements in the template or modifies their structure. Consider them as instructions for Angular to manipulate the Document Object Model (DOM).

Types of Directives

  • Component Directives
  • Structural Directives
  • Attribute Directives
  • Custom Directives

Component Directives

A component directive in Angular is a TypeScript class decorated with Component decorator that combines HTML templates, CSS styles, and business logic to create reusable UI elements. Components are the fundamental building blocks of Angular applications that control specific parts of the user interface and handle user interactions. Component Directives are directives in Angular that control a portion of the UI by using their own template, logic, and styles. Every Angular component is essentially a component directive.

KeyCharacteristics

  • Encapsulated: Components have their own scope with isolated styles and data
  • Reusable: Can be used multiple times throughout the application
  • Interactive: Handles user events and manages component state
  • Hierarchical: Components can contain other components forming a treestructure
  • Lifecycle-aware: Has built-in lifecycle hooks for initialization and cleanup

Core Elements

  • Component Decorator: Contains metadata like selector, template, and styles
  • Selector: CSS selector that identifies how to use the component in templates
  • Template/TemplateUrl: Defines the HTML structure (inline or external file)
  • Styles/StyleUrls: Component-specific CSS styling (inline or external file)
  • Class Properties: Store component data and state
  • Class Methods: Handle component logic and user interactions

Lifecycle Hooks

  • ngOnInit(): Called after component initialization, ideal for setup logic
  • ngOnChanges(): Called when input properties change
  • ngAfterViewInit(): Called after component's view is fully initialized
  • ngOnDestroy(): Called before component destruction, used for cleanup

Components serve as the foundation of Angular applications, providing a structured way to build interactive and maintainable user interfaces while promoting code reusability and separation of concerns

Component itself is a directive, so any component is a example to it

Structural Directives

Structural directives are Angular directives that change the DOM layout by adding, removing, or replacing HTML elements. They are responsible for shaping and reshaping the DOM structure based on certain conditions or data. Structural directives are easily identified by the asterisk (*) prefix and work by manipulating the template structure rather than just modifying element properties.Unlike component directives, structural directives don’t have their own template or styles. It is shorthand for Angular’s template syntax (ng-template).

Key Characteristics

  • DOM Manipulation: Add, remove, or replace elements in the DOM tree
  • Asterisk Syntax: Always prefixed with * (e.g., *ngIf, *ngFor)
  • Template Transformation: Convert the asterisk syntax into >ng-template> elements internally
  • Conditional Rendering: Control which parts of the template are rendered
  • Performance Impact: Directly affect rendering performance by changing DOM structure
  • Single Usage: Only one structural directive can be applied per element
